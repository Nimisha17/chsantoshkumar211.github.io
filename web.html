Web Security Training plan

1) Importance of learning development:

Before starting to focus on web security, learn to develop a web application from scratch using one of the development languages. This will really help you along your way.

A person who knows how to develop things from ground can learn to break things effectively !

Learn any of the development frameworks (I used to learn in PHP but it’s outdated now). You can use any of the following below:

  • Python Django
  • Ruby on Rails
  • NodeJs
  • PHP

There are several others but I strongly suggest you start with one of the above. Learn it properly, how to develop complicated apps from ground using one of the above frameworks. The app should be a sort of an e-commerce platform. For the front end, there are several templates available online which you can reuse. Don’t waste too much time on front-end rather focus on the functionalities and back-end.

The Web application should have the following functionalities:

  • Login page using PostgreSQL/MySQL in back end
  • Products should be displayed in home page
    • Title, where name should be displayed
    • Description if possible
    • Button for adding into cart
  • Interactive page using JQuery and JavaScript (to buy products)
    • User should be able to add products to the cart
    • Delete items from the cart
    • Checkout page
  • Profile page to display the details of the user
  • A logout button

This will help you a lot.

3) How to approach a problem:

There is a methodology is testing/recon a web challenge which can easily be automated in python. I have already written a script for this but I am not sharing it because I want all of you to write your own scripts for the same. Here are the steps:

  • Look into the common places of interesting information. Some good places to check are:

    • HTML source code
    • Robots.txt
    • HTTP response headers
    • Cookies

Analyse these details to find out anything interesting is there. Most of the times, you will get interesting things out of these. Is there a hint in the source ? Is the application hiding some sensitive directories from search engines in robots.txt ? If cookies are encoded (base64 ?) then try decoding to find any juicy stuffs out.


  • Before trying to exploit, see what the challenge is doing and understand it. Ask yourself these questions about the challenges:

    • Is there any login/Register functionality ? Can we register a new account and login ? See what all functionalities can we do after logging in. Is there a password reset ? Can we send message to admins/other users ?

    • What is the main functionality of the challenge ? What is it trying to do ? In most cases, we need to abuse/exploit the main functionality to get flag.


  • If you get answers for all the above questions, now you know precisely how the application works. Now let’s try to exploit it:

    • Identify all the places of userinput (login/register/password reset pages ?)

    • Identify all GET/POST parameters which we can control. Identify the context of user inputs and parameters.

    • Try exploiting it according to the context.

      • For ex: a login page, most probably, there will be a database behind which fetches the username/pass. So if vulnerability exist, it would most often be database exploitation or in other words sql injections !

      • If it is a message sending functionality, most probably the message you send will be read by another user or an admin right ? So what if we can include javascript statements in our message which might get triggered when admins/users read the message ? or in other words, the possibility of an XSS is high here.


  • Similarly understand all functionalities + contexts and see which vulnerability can occur with most probability. Your calculations will become more accurate when you solve more problems !

Do not blindly test applications without following the above steps. Learn and continue your each steps. Fuzzing with random payloads is not the way to play CTF.


4) Other very important things to do:

Technical skills are not the only important skill that you should have. Socialising with Good security researchers is very essentials for your growth. Other fellow researchers should know that you are also a passionate security guy who works hard. This will help you in getting good jobs, recognition etc. Make sure you follow whatever I said below which will help you get more publicity in the security community.

  • Twitter account:

    • Make sure you have a Twitter account. Most of the security researchers hangs around in twitter. Eg: Look at my twitter profile: a0xnirudh

    • Tweets should look good with valid hashtags + precise details. Use URL shortners like goo.gl to shorten big urls before sharing. Eg: 0daylabs - My blog twitter account.

    • Retweet others tweet only if you believe it’s highly relevant. Don’t retweet simply without any logic !

    • Security-Bugbounty-CTFs: A twitter list that I am maintaining which contains the best security researchers who regularly tweets. Make sure you follow this list everyday. I repeat, everyday ! Keep this as your routine.


  • Personal Website:

    • Create a good personal website in wordpress/github pages (I recommend github pages). You can learn about publishing a github page. Dbyll and Hyde is a very good personal jekyll themes to use.

    • Managing a blog on your github will increase the number of contributions on your github profile too.

    • Write technical contents only. No personal bullshits ! If you want to have write daily routine stuffs, create another blog (Why should others read your personal day to day things in the first place ? )

    • The main things you can write in your blog which the security community reads will be:

      • CTF writeups (Very important)

      • Bug Bounty writeups (only quality bugs, no simple stuff that an automated scanner can find)

      • Paper critiques (Read quality papers and put a write up)

      • Tutorials (especially on hard to understand topics)

    • When you are writing articles/writeups, make sure its clear and crisp. People should understand:

      • How you approach the problem. What are the initial recon steps

      • How did you identified the vulnerability ? What was the payload you used ?

      • How did you exploit it ? Detailed step by step with code blocks and images.

  • Give/Attend talks in conferences:

    • Attend conferences like c0c0n, Nullcon, GroundZero every year. If possible try to give a talk in any of the above conferences:

      • c0c0n CFP opens usually in March/April and happens in August.

      • Nullcon CFP opens in November/December and happens in Feb (next year)

      • Groundzero CFP opens in July/August and happens in November.. This sometimes happens in colombo too.

    • Try to attend all the above 3 conference every year. The best way is to apply for CFP and if you get selected, they will sponsor you. Conferences like Nullcon will give you a VIP pass if you just apply for CFP (even if it you get rejected)

    • The Primary Objective of attending these conferences is to get contacts of as many security researchers as possible. This would really help you in future ! If you don’t plan on socialising with others, never mind going to any of the conferences above. It’s useless !

  • IRC:

    • IRC is yet another good place where researchers hang around. There are some good channels in freenode which includes:

      • ##security
      • #vulnhub
      • #styprchall

    • Some really awesome channels are there in irc.smashthestack.org which is highly recommended:

      • #websec (some really awesome people here)

    • Hang around, if possible help others, listen to what kind of question others ask and how they reply etc.


  • Open source Tools:

    • Write automation scripts which can make life easier for both you and others. Make it open source so that others can use and contribute to it as well.

    • Try contributing to other good tools out there. OWASP tools is a great starting point but there are lot more !
